  • 我的「七遍学习法」：从接触到真正掌握一个新领域

    Jan 13, 2026

    在学习一个全新领域时，我慢慢发现一个事实：

    真正的难点，往往不是信息不够多，
    而是如何把零散的信息，变成自己真正能用的能力。

    在不断学习、实操和教学的过程中，我逐渐形成了一套相对稳定的方法。 它不追求“学得快”，而是关注是否真的掌握。
    我把它称为 —— 「七遍学习法」。

    第一遍：拉通整体流程，先看清全局

    第一遍，我只做一件事：看全局。

    这一遍不纠结细节是否完全理解，而是快速弄清楚：

    • 这个领域主要解决什么问题？
    • 整体流程大概是怎样的？
    • 安装、部署、运行分别处在哪个阶段？

    就像先看地图，而不是一上来就研究每一条小路。
    目标只有一个：知道自己身处哪里，要往哪里走。

    第二遍：修正文稿，把关键细节补齐

    有了整体轮廓之后，开始第二遍：填充细节。

    我会一边查资料，一边整理和修改文档，重点关注：

    • 每一步为什么要这样做
    • 参数、配置和概念之间的关系
    • 常见问题和容易踩的坑

    这一遍，知识会从「看过」慢慢变成「理解过」。

    第三遍：严格按文档，从零完整实操一遍

    第三遍，是对文档的一次真实检验。

    我会假设自己什么都不记得，只按照文档，从零开始完整跑一遍流程。

    任何需要“想当然”的地方，
    任何犹豫、出错或不顺畅的步骤，都会被记录下来并修正。

    这一遍通常问题最多，但也是文档提升最快的一遍。

    第四遍：录制教学视频，倒逼结构化理解

    当流程已经比较顺畅后，我会尝试录制一遍教学视频。

    讲给别人听，会自然暴露很多问题，比如：

    • 哪些地方其实自己并没想清楚
    • 哪些步骤逻辑是跳的
    • 哪些概念一说出口就卡壳

    很多时候，能讲清楚，比能做出来更难。

    第五遍：剪辑视频，再次审视细节

    剪辑视频，本身就是一次高密度复盘。

    在反复回放中，我经常会发现：

    • 表达不够严谨、容易误解的地方
    • 节奏过快、信息密度过高的片段
    • 本可以更简单的流程

    这一遍，让内容更精炼，也让理解更深入。

    第六遍：回看成品，站在学习者视角检查

    视频完成后，我会完整回看一遍，并刻意切换到学习者视角：

    • 是否缺少必要的前置说明？
    • 是否存在逻辑跳跃？
    • 是否需要补充背景或对比解释？

    这一遍，是对“教学效果”的最终检查。

    第七遍：脱稿应用与场景迁移，让知识真正变成能力

    前六遍，更多是在解决「会不会」；
    第七遍，验证的是「能不能用」。

    我会刻意脱离原有文档和视频，把知识用到新的场景中，例如：

    • 不看笔记，在另一套环境中重新搭建
    • 修改示例，解决一个真实需求
    • 即兴回答他人的问题，或当场讲解
    • 写一篇不依赖原文档的总结或踩坑记录

    只要在陌生环境和新问题下，仍然能顺利完成任务，
    基本可以确定：这项知识已经真正内化了。

    写在最后

    「七遍学习法」的核心，并不是简单地“多学几遍”，
    而是通过 输入 → 实操 → 输出 → 迁移，一步步逼近“掌握”这件事。

    如果你也经常学习新技术、新工具或新领域的内容， 不妨把「学会」当作中途站，把「能自由使用」作为终点。

  • 工作分水岭

    Oct 31, 2025

    做了管理者，发现做的好与做的不好的同事的两类特点

    做的好的同事，当遇到工作问题时，他们会想解决办法，会向我申请资源和决策支持

    做的不好的同事，当遇到工作问题时，他们会直接将问题抛给我来处理，稍微好一点的会问我怎么做，但并没有深度的思考探索

    我之前是直接帮他们处理并解决，我渐渐意识到每次帮他们处理后，我做了救火队员，他们会没有进步，团队的创造力随着时间推移渐渐萎靡

    这也间接的反映了两类管理者，一类是之前的我，直接帮同事处理问题，另一类是将来的我，引导同事找方向，给到工作方法。

    这两类倾向，常常和经验、薪资层级有关；但关键不在背景，而是思维习惯。

  • 培训经历

    Oct 14, 2025

    今天面试了一个培训的人，26 岁，他是达内培训机构出来的，在达内交了近两万的培训费用。以前是做的贷款销售，一个月 4 千左右在成都，我仿佛看到了当年的自己。

    回想 9 年前，我也经历了 Java 培训，交了 16800 块，分期。

    当时的境遇特别需要一份入门的工作，又没有什么存款，算是比较窘迫了吧。

    现在看着一份份投递过来的培训简历，很明显的简历包装，想到了自己以前的过往，心想还是给他们一个面试机会，让他们试试吧，说不定能改变某人的命运呢。

    还有一个尴尬的情况，公司需要来了能干活的人，因为公司也需要生存，暂时无法留有余地去培养经验不丰富的人。经验不丰富的人又需要工作机会来积累经验，形成了冲突。

    站在不同的立场上，也因为他的面试情况不佳，我没有通过他的面试。做出这个决定时，我心里还是有些遗憾的，我理解他的焦虑，也理解公司的现实——但现实终究不是怜悯能改变的。

    我想机会还是要给内心真正喜欢技术和这个行业的人，而不是谋求生存而进入该行业的人。

    也许每一次面试，都是一场命运交叉的瞬间。有人求生，有人求梦，而我，只希望能遇见那个为热爱而坚持的人。

  • 今天我离职了

    Mar 19, 2024

    今天我从公司离职了，在这个公司外包工作了一年。

    之所以是外包，是因为当时学历 ( 高中[大专在读，现 2024 已毕业，自考本科中] ) 不够要求，无法进入到公司 ( 中国电子，子公司 ) 内部成为正式员工，所以只能以外包的形式进入。

    如何进入此公司的？

    因为我和现在公司的大多成员都是上家公司 ( 电科云，子公司[已破产] ) 共事的同事。 大家对我的技术和工作能力的认可，其次是对平台的熟悉程度较高 ( 上家公司的公安政法业务转到了这边进行维护开发 ) 。

    为何离职？

    因为长期拖欠工资 ( 8 个月[包含现在的 3 月] ) 而离职，走了被迫离职的方式，进行了劳动仲裁。 在上家公司也是因为拖欠工资 ( 6 个月 ) 而被迫离职，我这两年相当于白打工了，感觉非常的惨。

    为什么坚持了这么久还不走？

    外包公司在入职早期的时候，正常发放了两个月工资，中期开始延迟发放零散的工资，后期 12 月发了一个月工资，勉强维持起日常开支。 我在年前就想走的了，但是 11 月发了部分和 12 月发了一次，就等着过春节会发放部分。结果未收到发放的工资，就有决心要离开了。 我和另外的外包同事也在积极沟通打款的事宜，一直拖着，说在走流程。至今也未能给到准确的时间打款，连公司正式员工也出现了拖欠工资的情况了，只发基本生活的工资 ( 两千多 ) 。

    在这个公司工作收获了什么？

    我想我在这个公司的收获是微乎其微的，大多是将上家公司积累的工作经验在这个公司应用。

    未来打算？

    随着 AI 技术的发展，想要投身到 AI 技术在 Web3 领域的应用中。

    为什么选择这个方向？

    AI 在未来是大势所趋，Web3 也将拥抱 AI，在国内限制的情况下，区块链技术我认为依然算是早期，将先进技术应用在 Web3 领域进行布道，是我很想做的事，并认为其仍有很大发展空间。

    我以前是做什么的？

    进入互联网行业也是误打误撞，找工作偶然通过培训机构 ( Java 培训 ) 进入的这个行业，并爱上了它。 早期有一年的 Java 开发经验和两年的 IT 系统管理经验。中后期开始接触运维相关的工作，有 5 年的运维管理经验。 现如今在学习区块链和 AI 相关的技术。

    安装

    hugo new theme acdiost-theme 创建主题后进入 cd themes/acdiost-theme 目录

    Install Tailwind CSS Install tailwindcss via npm, and create your tailwind.config.js file.

    npm install -D tailwindcss
npx tailwindcss init

    Add the paths to all of your template files in your tailwind.config.js file.

    /** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./layouts/**/*.{html,js}"],
  theme: {
    extend: {},
  },
  plugins: [],
}

    Add the @tailwind directives for each of Tailwind’s layers to your main CSS file.
